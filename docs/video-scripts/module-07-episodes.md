> 2. **当前目录** - 只对当前目录可见

### [4:00-7:00] 权限管理

**视觉元素**：
- 4 种配置方式对比
- 设置示例（settings.json, CLI flags）
- 权限选择决策树

**内容**：
> Claude Code 有精细的权限控制，让你安全地使用 AI。
>
> **4 种配置方式**：
> 1. **交互式提示** - "Always allow" 复选框
> 2. **/permissions 命令** - 运行时管理

> - **其他 Bash**：根据需要

### [10:00-14:00] 黄金工作流

> **黄金工作流**：这是 Boris 推荐的标准工作流程。
>
> **阶段 1: 探索** (Explore)
> - 使用子代理调查问题
> - 阅读相关文件
> - 理解代码库结构
>
> **阶段 2: 计划** (Plan)
> - 使用 Plan Mode (Shift+Tab 两次)
> - 分解任务为小步骤
> - 用户批准后继续
>
> **阶段 3: 编码** (Code)
> - 按计划实施
> - 小步迭代
> - 每步测试
>
> **阶段 4: 提交** (Commit)
> - 频繁提交
> - 把提交当作"保存点"
>
> **TDD 工作流**：
> 1. 先写测试
> 2. 确认测试失败
> 3. 让 AI 实现代码
> 4. 确认测试通过
> 5. 提交

**屏幕录制**：
- 完整演示黄金工作流

### [14:00-17:00] 多 Claude 工作流

- 工作流对比

**内容**：
> **多 Claude 工作流**：同时进行多个独立的 AI 任务。
>
> **使用 Git Worktrees**：
> - 创建独立的分支工作区
> - 每个 Claude 实例在自己的 worktree 中工作
> - 互不干扰，并行处理
>
> **场景示例**：
> - Worktree 1: 修复 bug
> - Worktree 2: 实现新功能
> - Worktree 3: 重构代码
>
> 命令：
> ```bash
> git worktree add ../feature-branch feature-branch
> ```
>
> 然后在两个终端中分别启动 Claude Code。

**屏幕录制**：
- 创建 git worktree
- 在两个 worktree 中并行工作

**内容**：
