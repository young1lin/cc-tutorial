#### 4. AAA 测试模式 (2 分钟)

**画面**: AAA 结构演示

**主持人**:
```
AAA = Arrange → Act → Assert

这是编写清晰测试的标准模式。

[代码对比]

❌ 不好的测试：
```typescript
it('works', () => {
  const user = new User();
  user.setAge(25);
  expect(user.age).toBe(25);
});
```
看不出哪是 Arrange、Act、Assert

✅ 好的测试：
```typescript
it('should set age when value is valid', () => {
  // Arrange
  const user = new User();
  const validAge = 25;

  // Act
  user.setAge(validAge);

  // Assert
  expect(user.age).toBe(validAge);
});
```

清晰的结构，一目了然！

记住：每个测试应该只有一个 Act。
如果有多个操作，拆分成多个测试。
```

#### 5. 常见 TDD 错误 (1 分钟)

**画面**: 错误清单

**主持人**:
```
常见的 TDD 错误：

❌ 错误 1：先写实现再写测试
→ 这不是 TDD，测试只是事后补充

❌ 错误 2：测试不够具体
→ "写一些用户验证的测试"
→ Claude 不知道具体要验证什么

❌ 错误 3：一次写太多测试
→ 测试和实现都变得复杂

❌ 错误 4：在实现阶段修改测试
→ "测试失败了，让我修改测试让它通过"
→ 失去了 TDD 的价值

记住：测试失败时，修改实现代码！
```

      - name: Run linter
        run: npm run lint

      - name: Run type check
        run: npm run type-check

      - name: Run tests
        run: npm test

      - name: Build
        run: npm run build
```

这个 CI 工作流确保：

- 所有测试通过
