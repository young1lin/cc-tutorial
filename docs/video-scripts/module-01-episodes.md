## Episode 1: Claude Code 简介

**章节**: Chapter 1
**标题**: 什么是 Claude Code？
**预计时长**: 10 分钟
**难度**: ⭐

### 脚本大纲

#### 1. 引入 (60 秒)

**画面**: 开场动画，标题卡片

**主持人**:
```
欢迎来到 Claude Code 教程！我是 [你的名字]。

你有没有想过：如果有一个 AI 助手，不仅能帮你写代码，
还能理解你的整个项目、制定实现计划、运行测试、修复 bug...
那会是什么样的体验？

今天，我们开始探索 Claude Code - 一个由 Anthropic 开发的
革命性 AI 编码助手。

在这一章中，你将了解 Claude Code 是什么，为什么它不同于
其他 AI 工具，以及为什么它正在改变软件开发的方式。
```

#### 2. Claude Code 是什么？(3 分钟)

**画面**: 动画展示 Claude Code 的工作方式

**主持人**:
```
那么，什么是 Claude Code？

简单来说，Claude Code 是一个高度自主的 AI 编程伙伴。
它不是简单的代码补全工具，而是一个能够：

[动画列出能力]
- 阅读和理解你的整个代码库
- 制定实现计划，而不只是直接写代码
- 编写、修改和重构代码
- 运行测试并修复失败
- 与 Git 和 GitHub 深度集成

最重要的是：它通过对话与你交互，就像和一个经验丰富的
开发者结对编程一样。
```

**画面**: 对比动画 - 传统 AI 助手 vs Claude Code

**主持人**:
```
让我用一个对比来说明：

传统 AI 助手（如 GitHub Copilot）：
- 你写代码时，它给出建议
- 你决定是否采纳
- 主要关注当前文件

Claude Code：
- 你描述目标，它制定计划
- 它理解整个项目结构
- 它可以执行多步骤任务
- 它可以运行命令和测试

这就是 Agent（代理）和 Tool（工具）的区别。
```

#### 3. 核心设计哲学 (3 分钟)

**画面**: Boris Power 的照片/引言

**主持人**:
```
Claude Code 有一个核心设计理念，这非常重要：

"AI 辅助工程" 而非 "AI 自动化工程"

这是什么意思？

[动画说明]
❌ AI 自动化工程：让 AI 完全自主，期望无人干预
✅ AI 辅助工程：将 AI 作为强大的伙伴，保持人类监督

正如 Claude Code 之父 Boris Power 所说：
"AI 编码助手是令人难以置信的倍增器，
但人类工程师仍然是节目的导演。"

这意味着：你始终保持对代码的控制和责任。
```

#### 4. 实际影响力 (2 分钟)

**画面**: Anthropic logo 和数据

**主持人**:
```
你可能想知道：Claude Code 实际上有多好用？

在 Anthropic 内部，Claude Code 被广泛采用。
事实上：约 90% 的 Claude Code 代码，
是由 Claude Code 本身编成的！

这意味着什么？
- Claude Code 是"自举"的
- 它的能力足以构建复杂工具
- 它被专业工程师信赖

知名专家也在使用：
- Zvi Mowshowitz 同时运行 5-10 个 Claude Code 实例
- Peter Bouda 说它"产生干净的、可测试的代码"
```

#### 5. LLM 的局限性 (1 分钟)

**画面**: 警告图标，严肃表情

**主持人**:
```
但有一点非常重要：Claude Code 不是银弹。

它本质上是"猜下一个 Token"的处理器。
这意味着：

如果你是行业专家，Claude Code 会放大你的能力。
但如果你一窍不通，它会撞得满头包。

[真实例子]
比如前端加载慢的问题：
- 专业工程师知道浏览器对同一域名只允许 6 个 HTTP 连接
- AI 只会一遍遍问"为什么这么卡？"

专业知识仍然不可或缺。
```

#### 6. 总结 (30 秒)

**画面**: 关键要点幻灯片

**主持人**:
```
让我们回顾一下今天学到的：

1. Claude Code 是高度自主的 AI 编码代理
2. 设计哲学是"AI 辅助工程"，人类保持责任
3. Plan Mode 是最重要的功能（我们下一章详细讲）
4. 需要专业知识，AI 放大而不是替代你的能力

在下一章中，我们将学习如何安装和配置 Claude Code。
```

## Episode 2: 安装与配置

**章节**: Chapter 2
**标题**: 快速上手 Claude Code
**预计时长**: 15 分钟
**难度**: ⭐

### 脚本大纲

#### 1. 引入 (45 秒)

**主持人**:
```
欢迎回来！在上一章，我们了解了 Claude Code 是什么。

今天，让我们把它安装到你的机器上，并完成基本配置。

即使你对命令行不太熟悉，也不要担心 - 我会一步步带你完成。
```

#### 2. 系统要求和准备 (2 分钟)

**画面**: 系统要求表格

**主持人**:
```
首先，确保你的系统满足基本要求：

[显示要求表格]
- 操作系统：Windows 10+、macOS、Linux
- 内存：4 GB RAM（推荐 8 GB）
- 稳定的网络连接
- Node.js 环境（可选）

最重要的是：你需要一个 Anthropic API 密钥。
让我们获取它。
```

#### 3. 获取 API 密钥 (2 分钟)

**画面**: 屏幕录制 - Anthropic Console

**主持人**:
```
首先，访问 console.anthropic.com

[录制：登录/注册过程]

导航到 API Keys 部分，点击"Create Key"。

重要：复制并保存这个密钥。你只会看到它一次！

[录制：创建密钥并复制]

现在，让我们设置环境变量。
```

#### 4. 安装 Claude Code CLI (3 分钟)

**画面**: 终端录制

**主持人**:
```
安装 Claude Code 非常简单。

如果你使用 npm：
[命令]
npm install -g @anthropic-ai/claude-code

如果你在 macOS 上，也可以用 Homebrew：
[命令]
brew install claude-code

让我演示一下...

[录制：安装过程]

安装完成后，验证一下：
[命令]
claude --version

看到版本号了吗？那就安装成功了！
```

#### 5. 配置环境变量 (2 分钟)

**主持人**:
```
现在，设置你的 API 密钥。

在 macOS 或 Linux 上，添加到 ~/.bashrc 或 ~/.zshrc：
[命令]
export ANTHROPIC_API_KEY="your-api-key-here"

在 Windows PowerShell 上：
[命令]
[System.Environment]::SetEnvironmentVariable(...)
```

#### 6. 首次运行 (3 分钟)

**画面**: 终端录制

**主持人**:
```
现在，让我们第一次运行 Claude Code！

导航到你的项目目录，然后输入：
[命令]
claude

[录制：首次启动向导]

Claude Code 会引导你完成初始设置。

如果向导没有出现，可以使用 /init 命令。
```

#### 7. CLAUDE.md 文件 (2 分钟)

**主持人**:
```
初始化过程中，Claude Code 会创建一个 CLAUDE.md 文件。

这是一个非常重要的文件！它是项目的"记忆"，
包含项目特定的信息和指导原则。

让我展示一个例子...

[显示示例 CLAUDE.md]

好的 CLAUDE.md 应该包含：
- 项目概述
- 技术栈
- 常用命令
- 代码风格指南
- 重要文件说明

保持简洁 - 这不是完整文档，而是给 Claude 的上下文。
```

#### 8. 权限系统 (2 分钟)

**画面**: 权限模式对比

**主持人**:
```
Claude Code 有一个安全的权限系统。

三种模式：
1. ask - 每次操作前询问（默认，推荐新手）
2. auto - 自动允许安全操作（适合信任的项目）
3. bypassPermissions - 完全跳过检查（仅用于自动化）

你也可以使用 /permissions 命令管理具体工具的权限。
```

#### 9. 总结 (30 秒)

**主持人**:
```
恭喜！你现在已经：
- 安装了 Claude Code
- 配置了 API 密钥
- 创建了 CLAUDE.md
- 理解了权限系统

下一章，我们将学习核心快捷键和基本用法。
```

**画面**: 快捷键演示动画

这是整个教程中最重要的快捷键：Shift+Tab

为什么它如此重要？因为 Plan Mode 让 Claude 先思考再行动。
我们将在下一章详细讲解。

记住这个快捷键 - 它是你最好的朋友！
```

Alt+V 让你从剪贴板粘贴图片到对话中。

使用场景：
- 粘贴 UI 设计稿，让 Claude 实现界面
- 粘贴错误截图，让 Claude 分析问题
- 粘贴架构图，帮助 Claude 理解系统

让我演示一下...

[录制：粘贴图片并让 Claude 分析]

非常强大！

双击 Escape 可以跳回历史记录，重新编辑之前的提示词。

[录制演示]

#### 6. 对话技巧 (2 分钟)

**主持人**:
```
有效的对话让 Claude Code 更好地帮助你：

DO（应该做的）：
✅ 提供上下文："阅读 src/api/user.ts，然后..."
✅ 具体化指令："创建一个名为 'avatarUrl' 的 prop"
✅ 分步骤执行："先做步骤 1，然后我们继续"

DON'T（不应该做的）：
❌ 模糊描述："添加一个功能"
❌ 缺少上下文："修复这个 bug"（哪个文件？）

- Escape Escape：跳回历史

基本命令：
- /help、/clear、/init、/permissions

下一章，我们将深入学习 Plan Mode。

## Episode 4: Plan Mode 革命

**章节**: Chapter 4
**标题**: Plan Mode - 最强大的功能
**预计时长**: 15 分钟
**难度**: ⭐⭐

### 脚本大纲

#### 1. 引入 (60 秒)

**主持人**:
```
这是模块 1 的最后一章，也是最重要的一章。

今天，我们深入学习 Plan Mode - Claude Code 的核心功能。

理解并善用 Plan Mode，将彻底改变你使用 AI 的方式。
```

#### 2. 什么是 Plan Mode？(3 分钟)

**画面**: 正常模式 vs Plan Mode 动画对比

**主持人**:
```
Plan Mode 让 Claude 在执行任何修改之前，先制定详细计划。

正常模式：
你 → Claude → 直接修改代码 ⚠️

Plan Mode：
你 → Claude → 制定计划 → 你审查 → 执行 ✅

这就像让一个建筑师先画蓝图，再开始施工。
```

#### 3. 如何使用 (2 分钟)

**画面**: 演示激活和退出

**主持人**:
```
使用非常简单：

按 Shift+Tab 一次 → 进入 Plan Mode
按 Shift+Tab 两次 → 退出 Plan Mode

或者使用命令：
/mode plan
/mode auto

[录制演示]
```

#### 4. 完整工作流 (4 分钟)

**画面**: 分步骤演示

**主持人**:
```
让我展示一个完整的 Plan Mode 工作流。

[录制：从开始到结束的完整流程]

步骤 1：激活 Plan Mode
[Shift+Tab]

步骤 2：描述需求
"添加用户个人资料页面，包括头像上传和编辑功能"

步骤 3：Claude 分析代码库
[显示分析过程]

步骤 4：Claude 展示计划
[显示详细计划]

步骤 5：审查和调整计划
"我想用不同的头像存储方案"
[Claude 调整计划]

步骤 6：批准并执行
[Shift+Tab 退出，Claude 开始实现]

步骤 7：完成！
```

#### 5. 何时使用 Plan Mode (2 分钟)

**画面**: 使用场景清单

**主持人**:
```
何时使用 Plan Mode：

✅ 应该使用：
- 多文件修改任务
- 不熟悉的代码库
- 复杂功能实现
- 大型重构
- 端到端功能开发

❌ 不应该使用：
- 快速的拼写修正
- 简单的代码格式化
- 非常熟悉的简单改动
```

#### 6. 阅读和修改计划 (2 分钟)

**主持人**:
```
审查计划时，关注：

1. 文件清单：是否完整？是否有遗漏？
2. 执行顺序：是否合理？是否有依赖问题？
3. 技术方案：是否适合项目？
4. 潜在风险：可能破坏什么？

不要害怕要求修改！这是 Plan Mode 的核心优势。
```

#### 7. 真实案例对比 (1 分钟)

**主持人**:
```
让我给你一个真实对比：

不使用 Plan Mode：
[快速演示]
- 直接开始写代码
- 30 分钟后发现缺少重要功能
- 需要大量返工

使用 Plan Mode：
[快速演示]
- 先制定计划
- 发现并补充遗漏
- 一次性完成，质量高

前期投入 5 分钟，后期节省 1 小时！
```

#### 8. 模块 1 总结 (1 分钟)

**画面**: 关键要点回顾

**主持人**:
```
恭喜！你完成了模块 1 的学习！

你现在已经掌握：
- Claude Code 是什么以及为什么重要
- 如何安装和配置
- 核心快捷键和命令
- Plan Mode 的强大功能

在模块 2 中，我们将学习核心工作流：
- "探索 → 计划 → 编码 → 提交"循环
- 测试驱动开发
- 多实例并行工作

继续你的学习之旅吧！
```
